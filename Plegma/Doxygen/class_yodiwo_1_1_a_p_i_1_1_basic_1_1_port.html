int Yodiwo.API.Basic.Port.IsInGraphs

summary>(readonly) returns whether this port is deployed in any graphs

Indicates the number of graphs this port is currently active in (i.e. number of deployed graphs that use this Port). On the Thing (Device) side it is meant to be read only. If 0, it means that the user has not created any stories which depend on and/or trigger this Port. As such the device should not send any PortEventMsg messages about it

string Yodiwo.API.Basic.Port.State
getset

Current (at latest update/sampling/trigger/etc) value of Port as String. Contains a string representation of the port's state, encoded according to the port's ePortType On receiving events the Cloud Server will attempt to parse the State based on its ePortType When sending events the Cloud Server will encode the new state into a string, again according to the Port's ePortType
