Detailed Description

Node Things Message Can be asyncronous (self-triggered) or in response to a ThingsReq request

If responding to a ThingReq, a ThingsMsg message should have:

  • ThingsMsg.Operation set to ThingReq's operation
  • APIMsg.ResponseToSeqNo set to ThingReq's APIMsg.SeqNo
  • ThingsMsg.Status set to True if ThingsReq was successfully handled and this Msg has valid data, False otherwise
  • if ThingsMsg.Status is True, ThingsMsg.Data set to correspond to requested Req's operation, set to Null otherwise

If an asynchronous message, then:

  • ThingsMsg.Operation is set to indicate what the receiving endpoint should do
  • APIMsg.ResponseToSeqNo is set to 0
  • ThingsMsg.Status set to True
  • ThingsMsg.Data set to correspond to requested Req's operation

Direction: bidirectional (Node->Cloud and Cloud->Node). Nodes can asynchronously trigger updates towards the Cloud with this message
